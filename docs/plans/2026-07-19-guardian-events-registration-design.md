# 守護活動與線上報名設計

## 目標

將使用者提供的三張正式海報放入「工會近期守護活動」，讓 2026 年 7 月 19 日的活動顯示為已結束，並讓未來活動可在 Vercel 正式站穩定完成線上報名。

## 活動內容

網站顯示四張活動卡：

1. `2026-07-19`「水泥叢林見金黃・松菸食農續飄香」，使用 `EDM1.jpg`，狀態為已結束，不可報名。
2. `2026-08-22`「志工勞服組特殊訓練」，使用 `62603.png`，開放報名。
3. `2026-09-20`「培訓第八屆・動保案件錄案員」，使用 `62607.jpg`，開放報名。
4. `2026-10-17`「動保領域信託課程」，共用 `62607.jpg`，開放報名。

使用者提供的 Instagram 影片截圖不使用。海報原檔放在 `public/events/`，避免外部圖片服務失效。

## 資料與 API 架構

目前正式站的 `/api/events` 回傳 404，且 `server.ts` 以本機 JSON 檔保存資料；Vercel 的靜態部署不會執行這些 Express 路由，也不能把函式執行環境的檔案視為持久化資料庫。

新架構使用既有 BeUnion Supabase 專案：

- `events`：活動內容、起訖時間、地點、講師、名額、已報名人數、海報路徑與報名狀態。
- `event_registrations`：活動、姓名、Email、電話、志工領域、備註與建立時間。
- `GET /api/events`：由 Vercel Function 讀取公開活動。
- `POST /api/events/:id/register`：由 Vercel Function 驗證輸入並新增報名。
- 資料庫約束負責防止重複報名；交易內鎖定活動列，避免同時報名造成超賣。

前端不會取得資料庫秘密金鑰。Supabase 公開資料表啟用 RLS：匿名訪客只能讀取活動，不能讀取任何報名者個資。報名寫入只經過受限制的 API／資料庫流程。

## 狀態與互動

活動狀態由活動結束時間與 `registration_open` 共同決定：

- 已結束：灰色標籤「活動已結束」，按鈕停用。
- 額滿：紅色標籤「額滿」，按鈕停用。
- 開放：顯示剩餘名額並可開啟報名表單。

送出報名時：

- 姓名、Email、電話必填並做格式與長度檢查。
- 同一活動同一 Email 不可重複報名。
- 已結束、關閉或額滿活動由伺服器再次拒絕，不能只依賴前端按鈕。
- 成功後顯示確認畫面並重新載入人數。
- 失敗時保留表單內容並顯示可理解的錯誤訊息。

## 驗證

自動測試涵蓋：

- 四場活動及三個海報路徑。
- 7 月 19 日活動顯示已結束且無法開啟報名。
- 必填欄位與 Email／電話格式。
- 正常報名成功並增加人數。
- 重複報名、額滿與已結束報名被拒絕。
- Supabase RLS 不允許匿名讀取報名者資料。

部署後直接檢查正式站 `/api/events`、活動卡片、海報回應，以及以專用測試活動完成一次報名／刪除驗證，避免污染正式活動名單。
